# Contributing to System Physics

This repository defines the System Physics layer, a proposed new infrastructure layer between observability and orchestration.

## What This Repo Is

A framework definition, not an implementation. It contains the mathematical model, validation methodology, and architectural specification for the system physics layer.

## How to Engage

**Challenge the thesis.** If you believe the layer isn't needed, open an issue and explain why. We want rigorous pushback.

**Challenge the math.** If the state variable definitions, conservation equations, or regime classification logic are wrong, open an issue with your analysis.

**Propose extensions.** If you see applications or domains where the framework should apply, open a discussion.

**Share incident data.** If you have anonymized incident telemetry you'd be willing to let us validate the model against, reach out. This is the single most valuable contribution at this stage. No event content needed, only aggregated metrics: rates, counts, latency distributions, error counts, and a timeline of what happened.

**Report where the framework is vague or hand-wavy.** If any claim in these documents feels like it's hiding behind impressive vocabulary rather than making a precise, testable statement, call it out. We would rather be specific and wrong than vague and unfalsifiable.

## What We Are Not Accepting Yet

- Implementation code (the engine is in private development and validation).
- Product feature requests (this repo defines the layer, not the product).
- Marketing language changes (the documents are written for engineers, not for sales).

## Standards

Be rigorous. Be honest. Be respectful. Attack ideas, not people. If you think something is wrong, explain why with evidence or reasoning, not with assertion.
